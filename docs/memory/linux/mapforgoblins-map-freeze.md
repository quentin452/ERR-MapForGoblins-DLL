**Core architecture facts:** markers are baked in `src/generated/goblin_map_data.cpp` (one file, both base area 60 + DLC area 61). Minor-dungeon entries carry a dungeon `areaNo`; ER has no map page for them, so they must be projected onto the overworld via `LEGACY_CONV` (baked in `goblin_legacy_conv.hpp` from the game's `WorldMapLegacyConvParam`). `apply_map_logic` (goblin_logic.cpp) sets each injected row's `eventFlagId` = the map-fragment gate. `require_map_fragments` gates icons on fragment discovery (default on → hid ~70-80% at game start; that, not missing data, was why it "looked broken").
