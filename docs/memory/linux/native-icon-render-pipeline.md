Goal: draw the GAME's real icons on overlay map markers (not the baked category atlas). Repo `<repo>`, branch `docs/worldmap-projection-re-brief`.

**SOLVED — texture pipeline (no game GPU bind, mod-agnostic), proven in-game:**
- Oodle **IAT hook** on `eldenring.exe` import of `oo2core_6_win64!OodleLZ_Decompress` (MinHook fails MH_ERROR_MEMORY_ALLOC — module >2GB; IAT hook works). Detour reads the decompressed buffer (in-process). Buffers are TRANSIENT (freed after ER parses) → we COPY out IN the hook. `.tpf.dcx`→TPF→headered DDS grabbed (`g_dds_list`); oodle block-decompresses ~256KB.
- Our texture manager (`goblin_overlay.cpp`): `create_tex_from_dds_mem` (raw DDS→our ID3D12Resource, BC native, no CPU decode→SRV), `copy_sheet_cached`/`copy_er_sheet_direct` (CopyResource a game sheet WHOLE → **sheet-as-atlas**, cached by src ptr → 1 ImGui draw call/sheet, UV sub-rect per icon). F1 dev: "Load ER image direct", DDS browser (`tpf_dds_count/at`).

**KEY finding (the wrong turn corrected):** ALL our markers carry **MAP-POINT iconIds** (`ITEM_ICONS.iconId`, `WORLD_MAP_POINT_PARAM.iconId`, the baked atlas) on the **1024×2048 worldmap sheet** (= the harvested grace sheet `g_grace_sprite.sheet`; verified icons match the live map 1:1). NOT inventory item-sheet iconIds. Proof: 351 ITEM_ICONS entries share iconId 381 (all armour→381 = generic armour MAP symbol). So the find-harvest path (`MENU_ItemIcon_<id>`, ensure_item_icon_srv) is a DIFFERENT inventory sheet → `native_item_icon` no-ops (harvested=false). `Marker::icon_id` (Step 1, populated by `item_icon_id(d.textId1)`) = a map-point iconId.

**SOLVED — map-point `iconId → rect` (RAM repo walk, commit 9f9a028, runtime-confirmed):** map-point icons (`MENU_MAP_<NN>` where NN = `WORLD_MAP_POINT_PARAM.iconId`, plus `MENU_MAP_ERR_*`/Church/...) live in the **FD4 image repo by-name tree `repo+0x80`** (`er+0x3d82510`) — NOT the `+0xb0` twin (= `MENU_MapTile_*`). Resident once world map opens. Added MENU_MAP_ branch to `harvest_repo_icons` (goblin_inject.cpp): rect = `img+0x74..0x80` (x0,y0,x1,y1), sheet = `[img+0x10]+0x70` → `store_map_icon_rect` into `g_map_icon_rects`/`g_map_icon_named`. Exposes `goblin::map_icon_rect(iconId,...)` / `map_icon_rect_by_name` / `map_icon_layout_count`. Auto-fills on map open — **no decompress, no sblytbnd, no force-load**. Sblytbnd Oodle/CSFile-hook avenue = DEAD END (decompressed BND4 freed after boot parse; not shipped). RE doc: `docs/re/windows_map_point_icon_layout_re_findings.md`.

Cross-validated: runtime `MENU_MAP_Church` rect `1660,150,1800,338` == offline `SB_MapCursor.layout` dump (`x1660 y150 w140 h188`). NB offline bake on **Linux is blocked** (`01_common.tpf.dcx` is Oodle/KRAK; `oo2core_6_win64.dll` won't load natively) — hence RAM walk is the path. `tools/probe_dcx.py` = author's offline DCX helper.

**⛔ "fill category_gpu_iconId per category" = DEAD END for item/feature categories (RESIDENCY, confirmed 2026-06-23 via [MAPICON-AVAIL] dump).** The per-category baked map-point iconIds DO exist in the sheet LAYOUT (381=armour, 405=stake, 394=pool, 404=spring, 406=map, 407=painting, 369=imp, 443=questNPC, 439=hawk, 438=interactable… one consistent iconId per category in goblin_map_data) — BUT the RAM repo walk only has a rect for symbols the game actually DRAWS, and **vanilla never draws item/feature map-pins** → iconIds 369–443 are NEVER harvested → `map_icon_rect(405)` returns nothing → nothing draws. The [MAPICON-AVAIL] enumeration showed ONLY iconId **1–89 + 208–261** resident (graces/dungeon-entrances/region/DLC/MP markers). Rects are an **irregular texture-pack** (e.g. 718,524 156×156 vs 1277,0 194×178) → can't extrapolate a missing rect from a grid. Full `SB_MapCursor.layout` (all rects) = sblytbnd asset data, **Oodle-blocked on Linux**. CONCLUSION: real ER map symbols are only available for features the vanilla map draws (≈ graces+bosses, handled by ERR's NAME-keyed `MENU_MAP_ERR_*` sheet). The mod's whole purpose is showing what the vanilla map does NOT (loot/items/features) → most categories have NO ER map symbol by construction. **Boss+Grace ≈ the ceiling.** Items already draw the real INVENTORY icon (`native_item_icons`) when resident; everything else stays the baked atlas (unavoidable). Higher-value adjacent work = improving inventory-icon coverage for the item categories that fall to a circle, NOT chasing map symbols. See [[overlay-icon-atlas]].
