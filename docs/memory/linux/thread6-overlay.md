Thread 6 = real in-game settings menu for MapForGoblins, replacing the F7/F8 hotkey-only toggles + INI-edit-restart DX. Chosen over the grace-ESD route (see [[grace-menu-esd-spike]]: ERR owns the grace talkesdbnd, re-ships it each update = merge too fragile) because reforged.dll proves a SELF-OWNED overlay (`gui_overlay::debug_gui`, WndProc/RegisterClass) runs STABLE in this ERR/Proton/vkd3d stack — EROverlay's crash was bad Present-sync, not Proton. User disabled EROverlay (crashes).

**Architecture:** ImGui panel drawn OVER the game framebuffer via a DXGI swapchain Present hook → floats over the 2D world map (not inside the game's map widget — impossible without heavy RE). All rendering INSIDE the Present hook on the render thread, using the game's OWN command queue captured from ExecuteCommandLists (the stability discipline). Answer to "show in the 2D map small/large": yes — gate visibility on map-open (`CSMenuMan+0xCD==7`, the retired detector documented in docs/ersc_hosting_and_map_autohide.md), small=corner widget ↔ large=full panel = ImGui window resize.

**Enabling facts (all confirmed in repo/env):** MinHook already vendored (modutils.cpp). CMake = FetchContent (added Dear ImGui v1.90.9 — classic imgui_impl_dx12 single-font-SRV Init). Cross-compile clang-cl+xwin HAS the DX12 SDK: headers `~/.local/share/xwin/sdk/include/{um/d3d12.h, shared/dxgi1_4.h}` + libs `lib/um/x86_64/{d3d12,dxgi}.lib`. C++ codebase → kiero-style hook is native.

**Phased plan:** P1 de-risk the hook (hello window, F1 toggle) ← SHIPPED, unverified. P2 map-open gating + small/large. P3 bind the 94 config fields live + Save-to-INI, COMBINED with the apply-layer unification (inject all categories, park disabled via areaNo=99 like the 7 sections already do via g_section_rows/apply_section_visibility — makes the 54 restart-only show_* toggles LIVE at zero map-open cost).
