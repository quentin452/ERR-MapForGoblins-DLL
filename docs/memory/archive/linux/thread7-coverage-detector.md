Thread 7 = live coverage-gap discovery for MapForGoblins (ERR map-icon mod, <repo>). Goal: while playing, detect when the player collects a PLACED world item the map has no marker for → log + in-game toast. Attacks the known coverage gaps (Equipment −196 etc.) without MapGenie diffing. Branch `feat/thread7-debug-unknown` (off feat/overlay). New module `src/goblin_debug_events.{hpp,cpp}`. See [[thread6-overlay]] (toast plumbing, overlay) + [[mapforgoblins-map-freeze]] (coverage audit) + [[mapforgoblins-map-open-freeze]] (SetEventFlag writer / unlock-all).

**Opt-in, both default false ([Debug] INI):** `debug_event_flags` (SetEventFlag hook) + `debug_item_grants` (AddItemFunc hook). Either activates; correlation needs BOTH. First runtime hooks the mod installs besides the overlay Present hook (mod was otherwise hook-free).

**Two hooks (AOBs lifted from Hexinton all-in-one CT v6.0; CT=XML at /tmp/hexinton-v6.ct — `<downloads>` + generic /tmp are OUTSIDE session working dirs so subagents get perm-denied; grep from MAIN thread).** Installed via modutils::hook in setup_mod (after marker-dump block); calls enable_hooks() again (MH_ApplyQueued) since setup's ran earlier. Self-disable on any AOB/hook/log fail.
- **SetEventFlag = `EventFlag_C1`** AOB `48 89 5C 24 08 48 89 74 24 18 57 48 83 EC 30 48 8B DA 41 0F B6 F8 8B 12 48 8B F1 85 D2 0F 84`. Sig `uint64_t(void* flagman /*rcx*/, uint32_t* flag_id /*rdx, id-by-ptr like IsEventFlag*/, uint8_t value /*r8b*/, uint64_t pad /*r9 safety*/)`. The `41 0F B6 F8`=movzx edi,r8b value-capture is what marks it the SETTER (getter lacks it). **BONUS: this is the long-sought in-game SetEventFlag WRITER** → unblocks runtime unlock-all (see [[mapforgoblins-map-open-freeze]]).
- **AddItemFunc** AOB `40 55 56 57 41 54 41 55 41 56 41 57 48 8D AC 24 70 FF FF FF 48 81 EC 90 01 00 00 48 C7 45 C8 FE FF FF FF 48 89 9C 24 D8 01 00 00 48 8B 05`. Sig `uint64_t(void* inv /*rcx*/, uint8_t* entries /*rdx*/, uint8_t* base /*r8*/, uint64_t count /*r9*/, uint64_t pad)`. **Entry layout CONFIRMED in-game: entry+0 = quantity, entry+4 = item id.** Item CATEGORY = id high nibble: 0x0=Armament, 0x1=Armour, 0x2=Talisman, 0x4=Goods, 0x8=Ash of War, else=item.
- Enemy defeat: NO dedicated hook in CT (only `Treasuredroprate`=drop-RATE RNG multiplier, useless). Named/boss kills set a flag → caught via the flag hook. Trash mobs (flagless) uncatchable, irrelevant.
- `safe_copy()` = SEH `__try/__except` (POD-only body, memcpy) guards EVERY game-pointer read (flag id + item entry) → bad ptr skips, never crashes; compiles under clang-cl.

**Pipeline (drain thread, off hot path):** flag detour timestamps each event → `{id,val,t_ns}` capped mutex inbox; item detour records `{t,item_id}` into a pruned ring + logs raw. Drain dedups (first-set-only), classifies: a flag is a gap candidate iff **collectible-shaped** (id≥1e9 AND flag-category `(id/1000)%10 ∈ {0,7}` — the categories the mod's own collect-flags use) AND **not** in `g_known_collect_flags` (built at init from all nonzero `textDisableFlagId1-8` + `clearedEventFlagId` across `MAP_ENTRIES`; MapEntry in src/generated/goblin_map_data.hpp). KEY INSIGHT: a placed lot's getItemFlagId IS its marker's textDisableFlagId, so collecting a placed item sets a flag the observer sees — no dedicated lot hook needed. Then **flag↔grant correlation**: hold candidates CORRELATION_WINDOW_NS=1.5s, report ONLY if a grant within ±1.5s AND not part of a ≥BURST_MIN(4)-flags/BURST_SPAN(100ms) burst (bulk-init backstop). On report → per-CATEGORY toast (nearest grant's item id → high nibble → gap_cat_toast_id). g_correlate = flag_on && item_on; flags-only falls back to looser "(uncorrelated)" report. Throttled summary `[correlation] reported N; suppressed N burst-init + M uncorrelated` (≥10s, only when a tally moves) makes suppression provable.

Per-ITEM name: resolution = NO RE (read GoodsName/WeaponName FMG via MsgRepository like marker names); dynamic DISPLAY = open (runtime FMG-entry rewrite of our row = no RE but widget may cache; or direct-string popup fn = RE).**

**CRASH ROOT-CAUSE FOUND + FIXED 2026-06-18 (commit 8945ab7).** A deterministic crash (0xC0000005 in ntdll heap allocator, same offset every time) hit at save-load (hooks on) and even at MAIN MENU (hooks off) — INDEPENDENT of the debug flags. NOT the debug hooks (disabling them only moved the crash load→menu); NOT the save (a clean restored backup still crashed). ROOT CAUSE: `inject_tutorial_popup_rows` (goblin_inject.cpp ~1444) sized the locator/data/wrapper arrays from `new_row_count` which was still hardcoded `4 + SECTION_COUNT*2` (=18, pre-P4), but P4 added 1 coverage-gap row + P4b added GAP_CAT_COUNT(6) per-category rows → all_rows held 25 → the write loop overflowed the HeapAlloc'd buffer by 7 rows → heap corruption → AV in ntdll at param-load (init-time, so flag-independent). Fix: `new_row_count = 4 + 1 + SECTION_COUNT*2 + GAP_CAT_COUNT`. **LESSON: when adding injected TutorialParam rows, update new_row_count to match all_rows pushes EXACTLY — it's the array-sizing source.** The crash dumper hardening (commit 8a24958: text triage w/ exception code+addr+module even when MiniDumpWriteDump leaves a 0-byte file; delete 0-byte dumps; prune to 4) is what pinpointed it (fault in ntdll, not MapForGoblins.dll/eldenring.exe → not our .text → buffer/heap bug).
