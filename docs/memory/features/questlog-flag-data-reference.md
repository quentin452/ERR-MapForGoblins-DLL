**`github.com/EldenRingQuestGiver/EldenRingQuestLog`** (author TheHaist, **MIT** — reuse/modify OK with notice). The Nexus "Elden Ring - Quest Log" mod (3091-adjacent, id 3585), open-source, same modding framework as our mod (`modutils`, `from/paramdef/`).

**Why it matters for [[thread3-grace-toggle]]'s sibling Thread 1 (quest-NPC layer):** `src/erquestlog_quests.hpp` (~3143 lines) is a hand-curated **quest → event-flag state machine for 34 NPC questlines** (base + DLC): alexander, ansbach, bernahl, blaidd, boc, corhyn, diallos, d, dungeater, fia, gowry, gurranq, hornsent, hyetta, igon, irina, jarbairn, kenneth, latenna, millicent, nepheli, patches, queelign, ranni, roderika, rogier, rya, sellen, seluvis, thops, trina, varre, ymir, yura. Each quest = ordered steps; each step gated by `esd_get_flag(<rawEventFlagId>, value=1)` or `esd_compare_inventory(<itemId>, n)` + a dialog-text id. e.g. Ranni = 11 steps (flags 1034509410/…/12019257); Kale alive/hostile/dead = 4700/4701/4703.

**The flag ids are raw ER event flags** → read at runtime via the **same IsEventFlag we already have** (goblin_kindling.cpp). So quest-state = NO new RE; this is the expensive curation, already done.
