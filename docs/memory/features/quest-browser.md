**Quest Browser = the agreed direction for quest-NPC navigation, replacing per-NPC map gating.** Why the pivot: quest-aware map gating (gate a WorldQuestNPC marker on its quest-active flag, see [[questlog-flag-data-reference]]) can only do ALL-OR-NOTHING per NPC — show every location when active, hide all when inactive. It can't show "the NPC where they currently are" because (1) our injected marker rows carry NO per-location gate flag (just icon/area/pos, eventFlagId=0), and (2) ER lets you start a quest at MULTIPLE locations so there's no single "first location" to fall back to (Goldmask's 4 rows: data-order puts his LATE Leyndell spot first, Outer Wall later). Per-location accuracy would need emevd spawn/despawn mining = the real v2.

**User's idea (2026-06-18, chosen):** instead of scattering NPCs on the map, a NEW ImGui menu that LISTS each NPC's quest steps IN ORDER; the player reads the step (location named in the text), goes there manually, "au bonheur la chance" if the quest is dead. Sidesteps the flag problem entirely.

**Q (2026-06-20): could live 3D NPC detection (WorldChrMan) be a quest cheat code? NO** — full verdict in [[live-param-vs-baked-data]]. WorldChrMan sees only LOADED NPCs near the player (world streams → far NPCs absent) and gives presence/pos/alive, NOT quest state/steps (EMEVD/ESD). Quest progress = event flags (already live-read). So the 3D doesn't replace static MSB placement + step text; only a local "nearby NPC" radar niche.

**MapGenie rejected** as the step/location source: proprietary commercial data (ToS forbids scraping) + its coords are MapGenie-space not ER-space (the Thread 2 drift). Used the MIT EldenRingQuestLog instead — already vendored for flags.

**TOOLING RELIABILITY (hard-won 2026-06-18, commit 803652d):** (1) errflags (the er-save-lib bin) is AUTHORITATIVE — it computes the char flag-region offset PER FILE. (2) flagdiff.py HAD A CRITICAL BUG: hardcoded offset 0x3689D, but the offset SHIFTS per save as earlier data grows (saw 0x3689D vs 0x36C0D same char) → it read wrong bytes → garbage flag ids (produced bogus Iji 1034502743 = false in every save). FIXED to query errflags for per-file offset + added --monotonic. (3) MANY ER flags are REGION/PROXIMITY-toggling — true when you're near the area, false when away (e.g. the captured 1034509302 went true@18:45→false@18:53→true@live). A real death flag is MONOTONIC (false alive, true forever after, at ANY location). (4) Prefer LOW/global flags (<10000) — region-independent, won't flicker with player location; the 10AABBCDDD map flags can toggle. (5) The in-game capture tool MISSES some flags (ER sets some via non-hooked routines) AND catches region noise — use it only to narrow; CONFIRM with errflags monotonic save-diff. METHOD: alive-save vs dead-save diff (fixed flagdiff) → --monotonic with 1-2 more dead saves taken ELSEWHERE → pick a low/global monotonic flag → errflags-verify it's false in early-alive saves (rules out generic counters).

**DEFINITIVE METHOD (the live grey-out reads the flag REGARDLESS of player location, so the fail_flag MUST be cross-location-persistent):** intersect TWO NPC-dead saves taken at DIFFERENT locations, then subtract an alive save. `flagdiff <alive> <dead-locA> --monotonic <dead-locB-far>`. Proximity flags (true only near the NPC's region) drop out; only globally-persistent death flags survive. CRUCIAL: even LOW flags (<10000) can be proximity-toggling — 558 looked perfect (monotonic across 3 Liurnia saves) but went FALSE once the player traveled far. Don't trust a flag until verified true at a FAR location. errflags-verify the survivor across 4-5 timepoints (false in all alive, true at every dead location incl far, set only after THIS NPC's death).

**FULL METHOD (when post-death travel adds noise):** (1) intersect TWO NPC-dead saves at DIFFERENT far locations → kills proximity flags; (2) intersect that with the in-game kill-window CAPTURE (flagcapture.txt) → kills post-death travel flags (graces/discovery that persist everywhere); (3) errflags-verify survivors are false in all alive saves + true at both far spots. Preserve dead saves to /tmp before ERR Backups overwrite them. For Seluvis this gave 5 → picked 1034509302 = seluvis_q99 (QuestLog-named, his own 1034509* namespace).
