The mod builds **4 profiles** (`err`, `vanilla`, `convergence`, `erte`), each with generated data
under `src/generated[_<profile>]/`. Only `err` is committed; the others are git-ignored and
regenerated. The data pipeline is **Windows-only** (needs pythonnet + Andre.SoulsFormats + the mod).

**This machine's setup (as of 2026-06-18):**
- Game **UXM-unpacked** (with SOTE DLC) at `<game_dir>`.
- `tools/config.ini` filled: `game_dir` = that path; `err_mod_dir=<windows_downloads>\ERR_mod`,
  `erte_mod_dir=<windows_downloads>\ERTE_mod`, `convergence_mod_dir=<windows_downloads>\Convergence_mod`. Each
  mod overlay was extracted from its `<windows_downloads>\*.zip` — only the subtrees the pipeline reads
  (`regulation.bin`, `map/MapStudio`, `event`, `msg`). Convergence/ERTE are partial overlays staged
  over vanilla by `prepare_merged_src.py`; ERR is a full overlay.
- **`oo2core_6_win64.dll`** copied into `tools/` AND `tools/lib/` (SoulsFormats needs it to
  decompress KRAK `.dcx`; it searches the cwd, so run with cwd=`tools/` when standalone).
- **mmap shim** at `<windows_temp>\mfg_aux\sitecustomize.py` — REQUIRED:
  the bundled Andre.SoulsFormats memory-maps temp files, so the scripts' `os.unlink` fails and
  reused pid-scoped temp names collide. The shim makes `os.getpid()` unique-per-call + tolerates
  unlink. Run the pipeline with `PYTHONPATH` pointed at that dir.

**Run a profile** (build.bat gates on VS2022 + has the cmd-banner bug, so call Python directly).
**Python gotcha:** use **`py -3.14`** (C:\Python314, has **pythonnet**) for any pipeline/MSB
script — bare `python` is a *different* 3.14 without pythonnet and fails. `py -3.13` is the
only one with **PIL** (for the icon PNG makers). Pipeline runs incrementally (per-stage
content-hash cache), so editing one generator only re-runs that stage + `generate_data`:
```
PYTHONUTF8=1 PYTHONPATH=<windows_temp>\mfg_aux MFG_PROFILE=<profile> py -3.14 tools/build_pipeline.py --profile <profile>
```

**Two real bugs fixed & merged (PR #1, master):** `extract_all_items.read_fmg_names` skipped the
DLC name FMGs (`GoodsName_dlc01.fmg`) → DLC items unnamed → name-based loot categories dropped them
(only non-ERR profiles, since ERR re-bakes names into the base FMG). And the Rune Arc filter

Coverage docs (`docs/coverage_*.md`) are generated by `tools/coverage_vs_mapgenie.py` — run it under
`PYTHONUTF8=1` on Windows (it writes emoji, crashes on cp1252). See [[windows-tooling-gotchas]].

**GFX bake** (`tools/build_vanilla_gfx.py` → `assets/menu/02_120_worldmap_{vanilla,convergence,err,erte}.gfx`):
needs **FFDEC** + **PIL**. FFDEC (JPEXS) is installed at `C:\Program Files (x86)\FFDec\ffdec-cli.exe`
(as of 2026-06-18; `ffdec_cmd()` now probes this path, so no `FFDEC_CLI` needed). PIL only lives in
**`py -3.13`** here (the default `py` = 3.14 has no PIL), so run bakes with `py -3.13 tools\build_vanilla_gfx.py --profile X`.
The overlay extraction into `<windows_downloads>\Convergence_mod` only staged the data subtrees the *data*
pipeline reads (event/map/msg/regulation.bin), **not `menu/`** — so the gfx bake first failed with
"base gfx not found". Fix: the Convergence base worldmap lives in the mod zip at
`ConvergenceER/mod/menu/02_120_worldmap.gfx` (overlay root = `ConvergenceER/mod/`); extract it to
`<windows_downloads>\Convergence_mod\menu\` (done 2026-06-18, now all 4 profiles bake). vanilla=erte base
(game vanilla, 348 frames, offset 0); convergence base = 756 frames (offset 408).
